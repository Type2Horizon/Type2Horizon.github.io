Trends in Startups - Aggregate Functions Code

Howdy! It’s your first day as a TechCrunch reporter. Your first task is to write an article on the rising trends in the startup world.

To get you started with your research, your boss emailed you a project.sqlite file that contains a table called startups. It is a portfolio of some of the biggest names in the industry.

Write queries with aggregate functions to retrieve some interesting insights about these companies.

  1. /* View complete startups table. */
  2. SELECT *
  3. FROM startups;
  1. /* Calculates the total number of companies in the startups table. */
  2. SELECT COUNT(*)
  3. FROM startups;
  1. /* Calculates the total value of all companies in the startups table. */
  2. SELECT SUM(valuation)
  3. FROM startups;
  1. /* Returns the highest amount raised by a startup. */
  2. SELECT MAX(raised)
  3. FROM startups;
  1. /* Returns the maximum amount of money raised, during ‘Seed’ stage. */
  2. SELECT MAX(raised)
  3. FROM startups
  4. WHERE stage = 'Seed';
  1. /* Returns the year the oldest company in the table was founded. */
  2. SELECT MIN(founded)
  3. FROM startups;
  1. /* Returns the average valuation. */
  2. SELECT AVG(valuation)
  3. FROM startups;
  1. /* Returns the average valuation, in each category. */
  2. SELECT category, AVG(valuation)
  3. FROM startups
  4. GROUP BY category;
  1. /* Returns the average valuation, in each category to two decimal places. */
  2. SELECT category, ROUND(AVG(valuation), 2)
  3. FROM startups
  4. GROUP BY category;
  1. /* Returns the average valuation, in each category to two decimal places and order the list from highest averages to lowest. */
  2. SELECT category, ROUND(AVG(valuation), 2)
  3. FROM startups
  4. GROUP BY 1
  5. ORDER BY 2 DESC;
  1. /* Returns each category with the total number of companies that belong to it. */
  2. SELECT category, COUNT(*)
  3. FROM startups
  4. GROUP BY category;
  1. /* Returns each category with the total number of companies that belong to it. Filter to only include categories that have more than three companies. */
  2. SELECT category, COUNT(*)
  3. FROM startups
  4. GROUP BY category
  5. HAVING COUNT(*) > 3
  6. ORDER BY 2 DESC;
  1. /* Returns the average size of a startup in each location. */
  2. SELECT location, AVG(employees)
  3. FROM startups
  4. GROUP BY location;
  1. /* Returns the average size of a startup in each location, with average sizes above 500. */
  2. SELECT location, AVG(employees)
  3. FROM startups
  4. GROUP BY location
  5. HAVING AVG(employees) > 500;
